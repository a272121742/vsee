# 函数助理

主要负责进行函数处理或转换，属于组合子范畴，虽然我们推荐直接使用函数式编程中的经典库`Ramda`和`lodash`，但我们仍然补充了一些常见的方法以弥补他们的不足。

对于函数流程的处理，我们分为：

- noop （无为）；
- always (恒定)；
- identity （维持，`Ramda`提供）；
- clone （克隆，`Ramda`提供）；
- or （选择）；
- fork （分流）；
- join （聚集）；
- map （映射）；
- reduce （规约）；

## gather(fn) => (argsArr) => {}

将原先发散的不定参数，聚合成收缩的单参数形式，例如：

```javascript
import { gather } from '@util/fn-helper.js';
// 例如
// 原函数`console.log`支持不定参数，现在我们来将参数收敛，类似`apply`，但不改变绑定对象
const log = gather(console.log);
log(['【警告】', '数据异常', 40001]);
const max = gather(Math.max);
max([1,2,4,3]);  // => 3
```

## spread(fn) => (...args) => {}

将原先聚合收缩但单参数形式，发散成展开的不定参数形式，例如：

```javascript
import { spread } from '@util/fn-helper.js';

// 原函数`Promise.all`接受单参数数组，不支持不定参数，现在我们来将参数展开，类似`call`，但不改变绑定对象
var promiseAll = spread(Promise.all);
promiseAll(promiseA, promiseB, promiseC);
```

## reverse(...args) => ()

反转原参数的调用过程，例如：

```javascript
import { reverse } from '@util/fn-helper.js';

const reverseLog = reverse(console.log);

console.log(1, 2, 3, 4); // => 1, 2, 3, 4
reverseLog(1, 2, 3, 4);  // => 4, 3, 2, 1
```

## defaultBy

`Ramda`提供了`defaultTo`函数，用于祛除不合法返回值（`null`、`undefined`、`NaN`），有时，为了能增加不合法数据的范围，此方法就不具备扩展性了。因此，我们提供了`defaultBy`方法， 可以定义一个返回`Boolean`的函数以实现自定义数据判断。

```javascript
import { defaultBy } from '@util/fn-helper.js';

// 判断是否为空对象
const isEmptyObj = (value) => Object.keys(value).length;
// 获取对象，如果是空对象，返回`null`
const getObj = defaultBy(isEmptyObj, null);
// 获取值
const infoObj = getObj(info);  // 如果info为空对象，则返回`null`，否则返回原对象
```

## tapClone

`tap`组合子的拓展，原`tap`组合子会拦截数据，但如果数据是引用类型，则会产生引用传递的问题。如果不希望发生引用传递，除了在给定参数时使用`clone`，更好方式是使用`tapClone`。可用于拦截器进行参数拦截（单个参数），相对于`tap`操作，其拦截的对象不会产生引用关系传递。

```javascript
import { tabClone } from '@util/fn-helper.js';
import axios from 'axios';

axios.interceptors.request.use(tabClone((clonedConfig) => {
  // 处理clonedConfig，而无需返回clonedConfig，对clonedConfig的所有操作不会影响最终返回值
}));
```

## alt

`alt`是`defaultTo`的函数版升级，先对参数做`targetFn`的取值判断，如果数据不合规范，则返回预定义函数`defaultFn`的值。

```javascript
import { alt } from '@util/fn-helper.js';

const getAge = (age) => {
  if (age < 100 || age >= 0) {
    return age;
  }
  return null;
}
const getDefaultAge = (age) => 18;

const getSafeAge = alt(getAge, getDefaultAge);

getSafeAge(34); // => 34
getSafeAge(-1); // => 18
getSafeAge(103); // => 18
getSafeAge(null); // => 18
```

> alt函子是可以通过`R.cond`函数转换得来的。

> `alt`等价于`(targetFn, defaultFn) => R.cond([[targetFn, R.identity], [R.T, defaultFn]])`

## altBy

`altBy`是`alt`的升级版，模式同`defaultBy`一样，会有额外的函数去判断返回值是否合规。

```javascript
import { altBy } from '@util/fn-helper.js';

const isSafeAge = (age) => (age < 100 || age >= 0);
const getAge = (age) => age;
const getDefaultAge = (age) => 18;

const getSafeAge = altBy(isSafeAge, getAge, getDefaultAge);

getSafeAge(34); // => 34
getSafeAge(-1); // => 18
getSafeAge(103); // => 18
getSafeAge(null); // => 18
```

不难发现，`altBy`相对于`alt`来说可以将函数拆解的更加粒度化。

> `altBy`函子是可以通过`R.ifElse`函数转换得来的。

> `altBy`等价于`(predicateFn, targetFn, defaultFn) => R.ifElse(predicateFn, targetFn, defaultFn)`

## seq

将参数按照函数队列进行同时执行。

```javascript
import { seq } from '@util/fn-helper.js';

// 设置对象属性
const convert = seq(
  (obj) => obj.id = parseInt(obj.id),
  (obj) => obj.date = obj.date.getTime(),
);
// 进行数据转换
convert({
  id: '123',
  date: '2020/01/12',
}); // => { id: 123, date: 1578758400000 }
```

## seqClone

将参数按照函数队列进行同时执行，所有参数均隔离其引用间的关系，并将返回参数结果按照原队列进行返还。

```javascript
import { seqClone } from '@util/fn-helper.js';

// 获取数组的最大值、最小值、中位数
const computed = seq(
  (arr) => Math.max(...arr),
  (arr) => Math.min(...arr),
  (arr) => arr[parseInt(arr.length / 2, 10)],
);
// 进行数据转换
computed([1, 2, 3, 4, 5]); // => [1, 5, 3]
```

## invokeCall

`invoker`函数的扩展，用于实现控制反转功能。方法签名如下：

```javascript
import { invokCall } from '@util/fn-helper.js';

invokCall('log', '消息', '-', '提示信息')(console);
```

## invokeApply

`invoker`函数的扩展，用于实现控制反转功能。方法签名如下：

```javascript
import { invokeApply } from '@util/fn-helper.js';

invokApply('log', ['消息', '-', '提示信息'])(console)
```
