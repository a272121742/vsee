# 别名路径

为了防止在深层路径文件中引入上层文件或资源所出现的`'../../../public/static/..'`这类问题，我们为开发者注册了诸多别名路径。这里，是开发者允许访问的别名路径：

```sh
- @          // 运行环境根目录
- @router    // 自动化路由
- @store     // 模块化数据存储中心
- @http      // 网络数据加载器
- @i18n      // 国际化中心
- @mock      // 数据模拟器
- @util      // 工具库
- @aop       // aop容器
- @dir       // 指令集
- @mix       // 混入器集合
- @static    // '@/public/static'的别名
- @comp      // 组件库
- ~          // 子项目根目录
- ~~         // 子项目中'common'文件夹别名
```

我们建议所有的开发者使用别名路径，而禁止使用类似`../..`的方式（`./`的方式可以继续使用）。使用别名能减少模块迁移而带来的不必要的麻烦。
